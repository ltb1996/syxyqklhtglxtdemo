# 前端与设计相关 Skill (技能) 列表

以下是本地可用的前端及UI/UX设计相关的Skill列表，它们可以帮助你在前端开发的各个环节中提升代码质量、用户体验和设计表现力：

| Skill 名称 | 中文名称 | 用途说明 | 适用场景 |
| :--- | :--- | :--- | :--- |
| **frontend-design** | **前端设计** | 创建独特、生产级别且具有极高设计质量的前端界面，生成高质量的前端代码，避免千篇一律的AI风格代码。 | 从零开始构建Web组件、页面、应用，或任何需要结合项目上下文的设计任务时。 |
| **typeset** | **优化排版** | 修复字体选择、视觉层级、大小、粗细和可读性，使文本排版更具设计感和意图。 | 当页面字体、文本层级看起来不协调，或者需要更精致、更具可读性的排版时。 |
| **teach-impeccable** | **教授完美设计** | 一次性设置工具，用于收集项目的设计上下文并保存到AI配置文件中，以建立持久的设计规范。 | 项目初始化或需要确立一致的前端设计指导原则时。 |
| **quieter** | **柔化设计** | 降低视觉上过于激进、嘈杂或刺激的设计，在保持质量的同时减少视觉强度。 | 当界面显得过于花哨、刺眼、拥挤，需要打造更平静、高雅的视觉体验时。 |
| **overdrive** | **突破常规设计** | 突破传统界限，实现技术上雄心勃勃的界面效果——如着色器、物理弹簧效果、滚动驱动展示、60fps动画等。 | 当你想制作令人惊艳、极具表现力、突破常规的超炫酷前端特效时。 |
| **polish** | **细节打磨** | 进行最终的质量审查，修复对齐、间距、一致性和微小细节问题。 | 上线前夕的预发布审查，需要解决界面看起来“哪里不对劲”，将设计从“良好”提升到“优秀”时。 |
| **optimize** | **优化性能** | 诊断和修复前端UI性能问题，涵盖加载速度、渲染性能、动画卡顿、图片优化和打包体积。 | 当用户反馈页面加载慢、卡顿、动画掉帧或打包体积过大，需要打造更流畅的体验时。 |
| **onboard** | **引导设计** | 设计和改进新用户引导流程、空状态（Empty States）和首次使用体验，帮助用户快速了解产品价值。 | 开发新功能、新页面，需要优化新用户的上手体验或添加空数据状态展示时。 |
| **normalize** | **标准化UI** | 审查并重新调整UI，使其符合设计系统的标准、间距、设计令牌（Tokens）和常见模式。 | 解决项目中存在的样式不统一、设计偏移，或想让某个界面的样式回归设计系统标准时。 |
| **harden** | **增强健壮性** | 通过更好的错误处理、国际化(i18n)支持、文本溢出处理和边缘情况管理，提高界面的弹性和健壮性。 | 让前端代码达到生产级别标准，妥善处理各种异常状态、文本超长和多语言场景时。 |
| **extract** | **提取与沉淀** | 识别出系统性复用的机会，提取并整合可复用的组件、设计令牌和模式到设计系统中。 | 重构前端代码，建立或丰富前端组件库，提取重复出现的UI模式为公共组件时。 |
| **delight** | **增加趣味性** | 添加令人愉悦的瞬间、个性化元素和意想不到的微交互，使界面令人难忘且使用愉快。 | 想为原本只注重功能性的界面增加动画、趣味交互和产品温度时。 |
| **distill** | **极简重构** | 通过移除不必要的复杂性，将设计精简到其本质，打造简单、强大且干净的设计。 | 当界面元素过多、显得嘈杂、需要精简以突出核心功能和降低认知负担时。 |
| **critique** | **设计评审** | 从用户体验（UX）视角评估设计，包含视觉层级、信息架构、认知负荷等，并提供定量评分和可操作的反馈。 | 需要对某个设计或组件进行专业UX评估、寻找改进空间，或希望获得设计修改建议时。 |
| **colorize** | **色彩丰富化** | 为过于单调或缺乏视觉趣味的功能添加战略性的色彩，使界面更具吸引力和表现力。 | 界面看起来灰暗、单调、缺乏活力，需要更有表现力或更温暖的色彩搭配时。 |
| **clarify** | **文本清晰化** | 改善不清晰的UX文案、错误信息、微文案、标签和说明，使界面更容易理解。 | 用户遇到令人困惑的提示、糟糕的错误信息或难以理解的操作说明，需要更好的UX写作时。 |
| **bolder** | **大胆设计** | 放大安全或无聊的设计，使其更具视觉冲击力和个性，同时保持可用性。 | 感觉当前设计过于平庸、保守、缺乏特点，希望增加视觉冲击力和性格时。 |
| **audit** | **技术审计** | 运行全面的技术质量检查，涵盖无障碍访问(a11y)、性能、主题、响应式设计等，生成带有P0-P3严重性评级的可操作报告。 | 需要对前端项目进行代码质量审查、无障碍检查、性能评估和反模式排查时。 |
| **arrange** | **布局调整** | 改善布局、间距和视觉节奏，修复单调的网格、不一致的间距和较弱的视觉层级。 | 当遇到页面布局感觉不对、间距拥挤、对齐有问题或想要更好的页面视觉结构时。 |
| **adapt** | **响应式适配** | 使设计适应不同的屏幕尺寸、设备、上下文或平台，实现断点、流式布局和合适的触摸目标。 | 开发移动端布局、解决跨设备兼容性问题、添加响应式断点时。 |
| **animate** | **动画增强** | 审查功能并增加有目的的动画、微交互和运动设计，以提升可用性和愉悦感。 | 当提及添加动画、过渡效果、悬停状态、微交互，或想让UI感觉“活过来”时。 |
